# X (Twitter) Integration Design

## Overview

Add X posting capability to the Cal Jakowski conspiracy bot. Each daily pipeline run posts AI-generated tweets alongside the existing Telegram dispatches. Tweets are written in Cal's voice and can include images from scraped story media links.

## Architecture

### New module: `src/x.js`

Handles all X API interaction using OAuth 1.0a signing with Node.js built-in `crypto` and `https` (no new dependencies).

**Exports:**

- `postTweet(text)` — post a text-only tweet, returns tweet ID
- `postTweetWithImage(text, imageUrl)` — download image from URL, upload via X media endpoint, post tweet with media attachment
- `verifyCredentials()` — calls `/2/users/me` to verify auth (used for smoke tests)

**Internal helpers:**

- `oauthSign(method, url, params)` — generates OAuth 1.0a Authorization header
- `uploadMedia(imageBuffer, mimeType)` — uploads to `https://upload.twitter.com/1.1/media/upload.json` (v1.1 media upload, required even for v2 tweets), returns media_id
- `createTweet(text, mediaIds)` — POST to `https://api.x.com/2/tweets`

### Config changes: `src/config.js`

Add to required env vars:
- `X_API_KEY`
- `X_API_SECRET`
- `X_ACCESS_TOKEN`
- `X_ACCESS_TOKEN_SECRET`

Add to config object:
- `xApiKey`
- `xApiSecret`
- `xAccessToken`
- `xAccessTokenSecret`

### Schema changes: `src/schema.js`

Add `x_posts` field to `storySchema`:

```js
x_posts: z.array(z.object({
  value: z.string(),
  value_citation: z.string().describe("Source URL for this value").optional(),
}))
```

These are short (max 280 char), cryptic, Cal-voice tweets generated by the AI alongside existing text_messages. Typically 1-3 per story.

### Character voice: conspiracy-caller skill

All X post generation MUST follow the character rules defined in `.claude/skills/conspiracy-caller`. Key constraints for tweets:

- **Tone:** sleep-deprived, anxious, sharp — not comedic, not satirical
- **Style:** short natural fragments, mid-thought starts, no clean intros or conclusions
- **Patterns:** "listen", "look", "I'm telling you", references to Ron, urgency without closure
- **Factuality:** anchor in at least one concrete fact from source, never invent facts, separate observation from speculation
- **Bad patterns to avoid:** sounding like a journalist, overexplaining, wrapping up neatly, inventing source facts

Tweet-specific adaptations:
- Even shorter than text_messages — fragments, not sentences
- Mix of caps and lowercase for emphasis
- Occasional "[deleted]" or "That wasn't there earlier." style paranoia posts
- 10% of posts should drive to caljakowski.com

### Scraper changes: `src/scraper.js`

Update the Firecrawl agent prompt to also generate `x_posts` per story. The prompt must include the character voice rules above so the AI generates tweets in Cal's authentic voice. Include a mix of:
- Cryptic observations (70%)
- Media references (20%)
- Site links to caljakowski.com (10%)

### Pipeline integration: `src/pipeline.js`

In `dispatchSend()`, after the Telegram sequence completes:

1. Pick one `x_post` from the story's `x_posts` array
2. Check if the story has `media_links` — if so, pick the first image-like URL
3. Call `postTweet()` or `postTweetWithImage()` accordingly
4. Log success/failure (non-fatal — X failure should not block Telegram)

Wrap in try/catch so X failures don't affect the existing Telegram pipeline.

### Tweet selection strategy

Each story has 1-3 x_posts. On dispatch:
- Use the first unused x_post
- Track which x_posts have been sent in the schedule entry
- This allows future expansion where remaining x_posts could be drip-posted throughout the day

## API Details

### Posting a tweet (v2)

```
POST https://api.x.com/2/tweets
Content-Type: application/json
Authorization: OAuth ...

{"text": "...", "media": {"media_ids": ["..."]}}
```

### Uploading media (v1.1)

```
POST https://upload.twitter.com/1.1/media/upload.json
Content-Type: multipart/form-data
Authorization: OAuth ...

media_data=[base64 encoded image]
```

Images must be < 5MB. Supported: JPEG, PNG, GIF, WEBP.

## Error Handling

- X API failures are non-fatal (logged, never block Telegram)
- Rate limit (429): log warning, skip this post
- Auth failure (401/403): log error with details for debugging
- Media upload failure: fall back to text-only tweet
- Image download failure: fall back to text-only tweet
- Oversized image: skip media, post text only

## Dependencies

None new. Uses Node.js built-in `crypto`, `https`, `fs`, and `Buffer`.
